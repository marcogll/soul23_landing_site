REGLA GENERAL N8N
1. Webhook recibe cualquier payload.
2. Switch por payload.type.
3. Si payload.email_report.templates.client existe y payload.email_report.enabled = true:
   - Renderizar template client con payload completo.
   - Enviar a payload.email_report.recipient_email.
   - From name sugerido: Talia | soul:23
   - Reply-To: talia@soul23.mx
4. Siempre enviar interno si existe payload.email_report.templates.internal:
   - Renderizar template internal con payload completo.
   - Enviar a hi@soul23.mx o al correo operativo que definas.
5. Si ratings_summary contiene score bajo o open_text_responses.sentiment = negative:
   - Crear alerta interna prioritaria.
6. Guardar todo en DB/Sheet/CRM con payload.type, metadata.source, submittedAt, email, telefono, tags.

NOTAS PARA N8N
- Si arrays llegan como arrays reales, convertirlos a texto para plantillas: join(", ").
- Si se usa Handlebars, registrar helper each y sanitizar valores.
- Para pre tags internos, se puede usar JSON.stringify(payload.answers, null, 2).
- El frontend no renderiza templates; n8n debe leerlos/copiar su HTML o usar estos paths como referencia.
- Si no hay VITE_S23_WEBHOOK_URL, el sitio hace console.warn y simula success; no llega a n8n.
- Todos los envios deberian incluir type y, cuando aplique, form_type.
